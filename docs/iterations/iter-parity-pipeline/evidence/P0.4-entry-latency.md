## 1. What shipped (no execution change)

`GET /api/backtest/analytics/reconcile?left&right` now returns `leftLatency` / `rightLatency`:
`{ matchedTrades, unmatchedFills, delaySeconds{median,mean,min,max}, delayBars{median,mean,min,max},
trades[]{orderId, proposedAtUtc, filledAtUtc, entryDelaySeconds, entryDelayBars, decisionTimeframe} }`.

- Per-trade latency = `TradeResult.OpenedAtUtc` (fill) − journalled `OrderProposed.OccurredAtUtc`
  (proposal, bar-open), **joined on OrderId**. `delayBars = delaySeconds / decisionTf.ToTimeSpan()`.
- Pure math: `EntryLatencyAnalyzer` (`Infrastructure/Reconcile`) — deterministic, no I/O.
- I/O: `LedgerReconcileService.BuildEntryLatencyAsync` (Web) streams the journal for `OrderProposed`
  events, loads `TradeResults`, hands both to the analyzer.
- NO cBot / kernel / adapter change (Q4 measure-first). Golden untouched → NO rebaseline.

## 2. Timezone quirk handled (R3-style)

The audit DB stamps cTrader `OccurredAtUtc` with a trailing `Z` (Kind=Utc) and tape without it
(Kind=Unspecified). The service parses with `AssumeUniversal | AdjustToUniversal` and the analyzer
subtracts on ticks (no timezone conversion), so both legs yield the true wall-clock delta.
`EntryLatencyAnalyzerTests.Timezone_suffix_mismatch_does_not_shift_the_delta` + the integration test's
mixed-Kind seed both prove it.
