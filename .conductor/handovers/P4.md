## Audit findings (s17)

### Bugs fixed (audit hardening)

1. **MaeMfeNormalizer lacked edge-case guards** — NaN/Infinity maePips/mfePips, zero pipSize, zero entryPrice, and zero stopLoss all produced silently wrong or crash-prone results. Fixed: all degenerate inputs now return `(null, null)`.
   - File: `src/TradingEngine.Services/Helpers/MaeMfeNormalizer.cs`
   - Tests added: `Normalize_NaNExcursions_ReturnsNull`, `Normalize_InfinityExcursions_ReturnsNull`, `Normalize_ZeroPipSize_ReturnsNull`, `Normalize_ZeroEntryPrice_ReturnsNull`, `Normalize_ZeroStopLoss_ReturnsNull`, `ComputeMaeR_NegativeStopDistance_ReturnsNull`, `ComputeMaeR_NaNInput_ReturnsNull`, `ComputeMfeR_InfinityInput_ReturnsNull`

2. **SqliteTradeRepository.SaveAsync lacked zero-price guard** — unlike `BackfillMaeMfe`, the save path would compute garbage MaeR/MfeR if entry or stop price were zero. Fixed: gate on `entry > 0 && stop > 0`.
   - File: `src/TradingEngine.Infrastructure/Persistence/Repositories/SqliteTradeRepository.cs:15-17`

3. **TypeScript types out of sync** — `TradeSummary` and `TradeDetail` interfaces in `api.types.ts` were missing `maeR`/`mfeR` fields that the backend DTOs now emit. Fixed: added `maeR?: number | null` and `mfeR?: number | null`.
   - File: `web-ui/src/app/models/api.types.ts`

## Weaknesses / shortcuts / assumptions

1. **F11 driven smoke NOT run** — The exploration funnel banner and exit-lab empty-state were not verified against a live running app with actual exploration-run data. The code paths are correct by structural inspection, but the end-to-end UX is unconfirmed. Next phase (P5) requires `run-shamshir` driven smokes per R6.

2. **F12 backfill endpoint NOT run against live DB** — No trade rows with `MaeR IS NULL` were verified to exist and be backfilled. The endpoint is well-structured but has never been called against a real database. Owner should run `POST /api/system/backfill-mae-mfe` on the production trading.db and verify `updated > 0`.

3. **MaeR/MfeR not displayed in any UI component** — The data flows all the way to the Angular types, but no table, chart, or detail component actually renders these fields. The pipeline to display them exists; only consumption is missing.

4. **TradeResult.MaeR/MfeR are domain fields stored ONLY in the DB write path** — If a TradeResult is constructed in code (e.g. by the kernel engine) and never persisted, the MaeR/MfeR will default to null. This is by design (they're computed for persistence), but it means in-memory analysis of domain objects won't have them unless the normalizer is called separately.

5. **Backfill uses `Math.Abs` for stop distance** — correct for long positions (entry > stop), but also works for shorts (stop > entry) because of `Math.Abs`. Verified in `Normalize_UsdJpy_SmallPipSize` test (short with stop above entry). This is correct.

6. **MAE/MFE pips are assumed non-negative** — The normalizer does not apply `Math.Abs` to maePips/mfePips. If a bug elsewhere produces negative excursion values, the computed MaeR/MfeR will also be negative, which is semantically wrong. The guard only rejects NaN/Infinity, not negative values. Decision: negative excursions indicate a deeper data bug; silently correcting with `Math.Abs` would hide it.
